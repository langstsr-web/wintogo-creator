# WinToGo Creator

Графическое приложение (PyQt6) для Linux, которое разворачивает **Windows 10/11
из ISO на внешний USB-накопитель** так, чтобы система загружалась с него на
реальном железе — аналог Rufus «Windows To Go» / WinToUSB, которых под Linux нет.

Работает через нативные инструменты Linux: `wimlib` для применения образа,
`parted`/`mkfs` для разметки, копирование EFI-загрузчика Windows. Установленная
Windows не является виртуальной машиной и загружается с настоящего железа —
поэтому не порождает VM-/RDP-признаков.

> ⚠️ **Статус: v0.1, ранняя версия.** Весь конвейер (безопасный выбор диска,
> разметка, форматирование, развёртывание образа, копирование EFI-загрузчика)
> реализован и протестирован. **Не завершена автоматическая генерация BCD**
> (хранилища конфигурации загрузки) — это единственная часть, которую нельзя
> проверить без загрузки на реальном оборудовании. См. раздел «BCD» ниже.

---

## Возможности

- Выбор ISO и **чтение списка редакций** (Home/Pro/…) прямо из образа.
- Список целевых дисков с **жёсткими предохранителями**:
  - системный диск (где `/`, `/boot`, `/boot/efi`) помечается ⛔ и запрещён к записи —
    проверка дублируется и в GUI, и в привилегированном ядре;
  - по умолчанию показываются только внешние/USB-диски;
  - подтверждение стирания требует **вручную ввести имя устройства**.
- Режимы **UEFI/GPT** (основной) и **BIOS/MBR** (экспериментальный).
- **Пробный прогон** (dry-run): показывает полный план команд, не трогая диск.
- Прогресс развёртывания и подробный лог.
- Повышение прав через **pkexec** (GUI работает без root; диск пишет отдельный
  привилегированный процесс).

## Зависимости

```bash
sudo apt install python3-pyqt6 wimtools gdisk dosfstools ntfs-3g \
                 libhivex-bin policykit-1 udisks2
```

| Пакет | Зачем |
|-------|-------|
| `python3-pyqt6` | графический интерфейс |
| `wimtools` | `wimlib-imagex` — применение install.wim/esd |
| `gdisk`, `dosfstools`, `ntfs-3g` | разметка и файловые системы |
| `libhivex-bin` / `python3-hivex` | генерация BCD (в разработке) |
| `policykit-1` (pkexec) | повышение прав из GUI |
| `udisks2` | монтирование ISO без root при чтении редакций |

## Запуск

```bash
python3 wintogo.py            # графическое окно
./install.sh                  # добавить ярлык в меню приложений
python3 wintogo.py --version
```

Порядок работы в окне: **1)** выбрать ISO → **2)** «Прочитать редакции» и выбрать
редакцию → **3)** выбрать внешний диск → **4)** параметры → «Создать загрузочный
диск». Рекомендуется сперва прогнать с галкой **«Пробный прогон»**.

## Что делает под капотом (UEFI)

1. `parted` → GPT: раздел **ESP** (FAT32, 300 МБ) + раздел **Windows** (NTFS, остаток).
2. `mkfs.fat` / `mkfs.ntfs`.
3. `wimlib-imagex apply <install.wim> <индекс> <раздел-Windows>`.
4. Копирование `Windows\Boot\EFI\*` (включая `bootmgfw.efi`) на ESP, плюс
   `\EFI\Boot\bootx64.efi` (фоллбэк) и шрифты boot-меню.
5. Создание **BCD** (см. ниже).

## BCD — незавершённая часть

Windows не загрузится без корректного **BCD** (`\EFI\Microsoft\Boot\BCD`). На
Windows его создаёт `bcdboot`, которого под Linux нет. В v0.1 приложение пробует,
по порядку:

1. **готовый шаблон** `assets/bcd/BCD` + `assets/bcd/BCD.json` (с GUID-плейсхолдерами) —
   патчится GUID целевого раздела чистым Python (без внешних зависимостей);
2. **сборку через `python3-hivex`** (пока `NotImplementedError` — точка доработки);
3. если не вышло — выводит **инструкцию** и помечает диск как «образ развёрнут,
   остался загрузчик».

**Как завершить диск вручную прямо сейчас** (пока BCD не автоматизирован):
загрузиться с установочной флешки Windows, на первом экране нажать **Shift+F10**
и выполнить (где `W:` — раздел Windows этого диска, `S:` — его EFI-раздел):

```
bcdboot W:\Windows /s S: /f UEFI
```

**Как сделать инструмент полностью «однокнопочным»:** один раз получить рабочий
BCD (например, с диска, сделанного Rufus, или выполнив `bcdboot` в Windows-VM),
положить его в `assets/bcd/BCD`, а рядом `assets/bcd/BCD.json` вида
`{"part_guid":"<GUID-раздела-Windows-в-шаблоне>","disk_guid":"<GUID-диска>"}`.
Дальше приложение будет патчить его под каждый новый диск автоматически.

## Ограничения и безопасность

- Приложение **перезаписывает выбранный диск целиком**. Всегда проверяйте выбор.
  Предохранители снижают риск, но окончательная ответственность — на пользователе.
- BIOS/MBR-режим экспериментальный (нужен `ms-sys` для загрузочного сектора NTFS).
- Split-WIM (`install.swm`) пока не поддерживается.
